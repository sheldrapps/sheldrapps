# PDF Cover Maker (PCM) - Creative Brief

## Purpose

Define creative, commercial, and localization strategy for Play Store listing assets of PDF Cover Maker (PCM).

This brief guides:

- Play Store copy generation
- feature graphic direction
- screenshot strategy
- regional localization
- golden listings
- bulk ficha generation
- conversion audits

This document is strategic source of truth for PCM fichas.

## Product Identity

### Working product name

PDF Cover Maker (PCM)

### Category

PDF utility / visual organization tool.

### What app really is

Tool to create a clean, custom visual cover for existing PDF file.

### What app is not

- Not full PDF editor
- Not OCR tool
- Not signing tool
- Not compression/merge/conversion suite
- Not internal page editing app
- Not advanced metadata manager

## Core Positioning

### Main positioning statement

Create better first impression for any PDF.

### Emotional promise

Make your PDFs look clean, personal, and worth opening.

### Functional truth

User can:

- pick PDF
- choose cover image
- adjust image (crop/zoom/rotate/preview)
- save new PDF copy

### Strategic balance

PCM positioning priority:

- visual result first
- speed of workflow second
- privacy/local processing as trust proof

## Conversion Promise

### Primary conversion promise

Create a custom PDF cover in seconds.

### Secondary conversion promise

Fast flow from file selection to polished result, without account or uploads.

### Trust proof

- 100% on-device flow
- no account required
- no cloud upload
- visual adjustment before save
- output as new copy

## Product Truth Guardrails

### Safe claims

- Make or create a PDF cover
- Improve first visual impression of a PDF
- Select, adjust, preview, save
- Keep processing on device
- Work without account/signup

### Claims to avoid

- "PDF editor" as primary claim
- "Edit all PDF content"
- OCR promises
- signature/e-sign features
- compression promises
- merge/split/conversion claims
- internal page editing claims
- advanced metadata replacement claims (unless explicitly supported)

### Wording rule (critical)

Primary brand language must use:

- "make a PDF cover"
- "create a PDF cover"

Do not use "change PDF cover" as core idea.
Can appear as secondary SEO phrasing only when useful.

## Core Conversion Pillars

Priority order for assets and copy:

1. PDF looks better first
2. workflow is fast
3. everything stays on device
4. no account, no uploads

## Screenshot Strategy

### Recommended screenshot order

1. Strong before/after: plain/blank PDF -> attractive custom cover
2. Simple flow: Select. Adjust. Done.
3. Visual control: crop, zoom, rotate, preview
4. Privacy trust: 100% on your device, no accounts, no uploads
5. Organization outcome: cleaner PDF library/list

### Visual principle

First screenshot must sell outcome, not settings.

## Final Creative Direction

### Tone

Clean, practical, modern, confidence-building.

### Core narrative

Not "edit PDF everything."
Yes: "create polished, custom cover that improves how PDF looks at first glance."

### Quality bar

Work is acceptable only if:

- copy and assets lead with "make/create a PDF cover"
- no unsupported all-in-one PDF claims appear
- screenshot 1 communicates immediate transformation
- privacy and no-upload trust proof is explicit
- localized angles feel native, not direct translation clones
