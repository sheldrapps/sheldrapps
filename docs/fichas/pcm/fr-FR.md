# Créer couverture PDF - fr-FR Play Store Listing

## Purpose

Create a Play Store ficha that clearly sells PDF cover creation, visual improvement, local processing, preview control, and safe save-as-copy behavior.

The listing should immediately communicate that users can create a better-looking cover for a PDF using an image they choose, adjust the layout, preview the result, and save a new PDF copy.

Créer couverture PDF est utile lorsque la première page d’un fichier PDF paraît vide, trop simple, peu élégante ou difficile à reconnaître.

Avec cette app, vous pouvez :

• vérifier la couverture finale avant l’enregistrement
• enregistrer une nouvelle copie du PDF tout en conservant l’original

Tout est traité sur votre appareil.
Sans compte.
Sans envoi de fichier.

Idéal pour les supports de cours, guides, documents de travail, rapports, notes, catalogues, portfolios et PDF personnels que vous souhaitez rendre plus clairs, mieux présentés et plus faciles à retrouver.

L’app se concentre sur une seule tâche : créer des couvertures PDF rapidement, simplement et en toute confidentialité.

## Market Angle Decision

Market angle selected:
Présentation soignée et simplicité.

Why this angle should convert in this locale:
For fr-FR, the strongest conversion angle is clean presentation with simple control. The copy should feel polished, practical, and refined, not overly technical. The user should immediately understand that the app helps make PDF files look clearer and more elegant without turning into a complex PDF suite.

Primary user desire:
Create a clean, attractive, and easy-to-recognize cover for a plain or unclear PDF first page.

Secondary trust proof:
Local processing, no account, no file upload, preview before saving, and saving as a new copy.

What should NOT be the first screenshot:
Settings screen, empty file list, technical PDF details, generic image editor UI, scanner-like screen, or privacy-only claim without showing the visual transformation.

Palette:

* secondary background: #252525
* accent: #B3261E
* optional support accent: #D4574F
* headline color: #FFFFFF
* subline color: #B8B8B8
* bullet/support text color: #E2D2D0

Reasoning:
For fr-FR, the visual system should feel refined, practical, calm, and PDF-native. A dark matte base with controlled PDF-red accents gives the listing a familiar PDF cue without making the design feel loud or aggressive.

The red accent should be used only as restrained emphasis: subtle depth behind the “after” PDF mockup, before/after indicators, small trust markers, thin lines, or controlled highlights.

General visual rules:
Keep the copy area dark, matte, and readable. The transformation must remain the main visual proof: plain PDF first page → polished custom PDF cover.

Avoid bright red glow behind text, white panels behind copy, busy document stacks, AI sparkle effects, cloud/upload symbols, EPUB/e-reader visuals, Kindle-like device positioning, scanner-like visuals, or anything that makes the app look like a general PDF editor.
