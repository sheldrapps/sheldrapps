# Criar capa para PDF - pt-BR Play Store Listing

## Purpose

Create a Play Store ficha that clearly sells PDF cover creation, visual improvement, local processing, preview control, and safe save-as-copy behavior.

The listing should immediately communicate that users can create a better-looking cover for a PDF using an image they choose, adjust the layout, preview the result, and save a new PDF copy.

## Market Angle Decision

Market angle selected:
Deixar PDFs mais bonitos e organizados.

Why this angle should convert in this locale:
For pt-BR, the strongest conversion angle is practical visual improvement. The copy should feel clear, useful, and approachable. The user should understand that the app helps make PDFs look better and easier to recognize without sounding like a complex office or PDF suite.

Primary user desire:
Create a clean, attractive, and recognizable cover for a plain or unclear PDF first page.

Secondary trust proof:
Processing on device, no account, no uploads, preview before saving, and saving as a new copy.

What should NOT be the first screenshot:
Settings screen, empty file list, technical PDF details, generic image editor UI, scanner-like screen, or privacy-only claim without showing the visual transformation.

* subline color: #B8B8B8

Reasoning:
For pt-BR, the visual system should feel practical, modern, approachable, and PDF-native. A dark matte base with controlled PDF-red accents gives the listing a familiar PDF cue without making the design feel aggressive or visually tiring.

The red accent should be used only as restrained emphasis: subtle depth behind the “after” PDF mockup, before/after indicators, small trust markers, thin lines, or controlled highlights.

General visual rules:
Keep the copy area dark, matte, and readable. The transformation must remain the main visual proof: plain PDF first page → polished custom PDF cover.

Avoid bright red glow behind text, white panels behind copy, busy document stacks, AI sparkle effects, cloud/upload symbols, EPUB/e-reader visuals, Kindle-like device positioning, scanner-like visuals, or anything that makes the app look like a general PDF editor.
