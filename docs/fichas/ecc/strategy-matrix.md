# ECC Strategy Matrix

## Purpose

Define the regional conversion strategy for EPUB Cover Changer across all supported Play Store locales.

This matrix guides:

* localized Play Store fichas
* feature graphics
* screenshots
* visual direction
* palette selection
* conversion audits
* bulk asset generation

The product truth must remain consistent across all locales:

EPUB Cover Changer replaces the cover of an EPUB file with a user-selected image, allows crop/layout adjustment, previews the result, and saves a new copy of the EPUB.

## Visual Generation Rules

Do not reuse the exact same color palette across all locales.

Keep a dark global brand base, but localize:

* accent temperature
* title contrast
* subline softness
* bullet visibility
* cover image style
* trust proof emphasis
* headline directness

`Background base` is the main canvas color.

`Background secondary` is used for:

* gradients
* cards
* panels
* mockup shadows
* subtle depth
* safe contrast behind devices

`Accent color` is used for:

* highlights
* thin lines
* active UI cues
* before/after arrows
* small glow zones
* short trust markers
* controlled visual emphasis

`Support accent` is used only when the asset needs a softer secondary highlight. It must not replace the main accent or dominate the background.

`Title color` must be used for the main headline only.

`Subline color` must be used for the subtitle or supporting line only.

`Bullet color` must be used for:

* bullets
* small proof points
* checkmarks
* short trust markers
* compact compatibility or privacy notes

## Background Production Rules

Every generated ficha must define the background with this level of specificity:

* dimension
* base color
* secondary color
* accent color
* accent placement
* safe area for copy
* texture allowed
* forbidden elements behind text

The copy zone must stay stable, dark, and readable.

Do not place bright panels behind title text.

Do not place large saturated red, orange, blue, or purple glows behind the headline.

Do not create dense patterns behind copy.

Do not use noisy gradients behind important text.

Do not place white shapes, bright book pages, or device highlights behind headline/subline copy.

Texture is allowed only if subtle and matte. Recommended maximum: below 3% visual intensity.

## Screenshot Strategy Rules

The first screenshot must sell the problem and transformation, not settings.

The visual sequence should follow:

1. Problem / before-after transformation
2. Simple flow / editor
3. Control / preview
4. Privacy / on-device trust
5. Saved result / safe copy / transformation proof

At least two visual assets should show either:

* a before/after transformation, or
* a clear final improved EPUB cover.

The first screenshot must be understandable in one glance at Play Store preview size.

Do not open with:

* settings
* empty library
* app menu
* technical file details
* dense editor controls
* generic cropper UI
* cloud/privacy claim without product context

## Feature Graphic Rules

The feature graphic must summarize the product in one glance.

It should communicate:

* EPUB cover replacement
* before/after transformation
* custom image result
* speed
* simplicity
* practical utility

It must not communicate:

* PDF editing
* generic document editing
* ebook reading
* ebook downloading
* DRM removal
* cloud upload
* AI cover generation
* template marketplace
* full EPUB editing

Recommended structure:

* headline in stable dark safe area
* plain/incorrect EPUB cover on one side
* polished replacement cover on the other side
* clear before/after relationship
* restrained accent behind the after result
* no bright glow behind the text

## Cover Image Rules

Preferred before-cover ideas:

* plain white generic cover
* incorrect-looking minimal cover
* ugly placeholder cover
* missing-cover style
* rough sketch cover
* title-only layout
* visually weak cover with poor spacing

Preferred after-cover ideas:

* polished illustration
* fantasy landscape
* cozy reading image
* pet photo
* travel memory
* public-domain inspired artwork
* aesthetic image that clearly improves the original

Avoid:

* copyrighted recognizable book covers
* official publisher artwork
* real brand documents
* fake Amazon, Kindle, Kobo, Apple Books, or Google Play Books UI
* covers that look too similar before and after
* tiny unreadable title text
* cluttered stacks of books
* AI-generation cues
* visuals that imply the app provides or downloads books

## Copy Rules

Use direct product language:

* change EPUB cover
* replace EPUB cover
* edit EPUB cover
* fix incorrect cover
* crop and adjust
* preview the result
* save a copy
* on-device processing
* no account
* no uploads

Localized copy should preserve the meaning, not translate mechanically.

The strongest global promise is:

Change the cover of any EPUB file in seconds.

The strongest global trust proof is:

Processed on your device. No accounts. No uploads.

The strongest global safety proof is:

Save a new copy of the EPUB.

## Claims to Avoid Globally

Avoid any claim or visual implication of:

* AI generation
* automatic cover search
* template marketplace
* DRM removal
* DRM bypass
* book downloads
* free books
* full EPUB editing
* EPUB text editing
* universal EPUB reader compatibility
* official affiliation with reading platforms
* cloud processing
* online sync
* publishing features
* format conversion unless explicitly supported in the current build

## Regional Differentiation Rules

Localization is not translation.

Each locale should adapt:

* problem framing
* tone
* visual warmth
* accent temperature
* after-cover style
* trust proof emphasis
* headline length
* copy rhythm

But every locale must preserve:

* EPUB cover replacement positioning
* before/after first impression
* local processing trust proof
* no account/no upload claim
* preview/control message
* save-copy reassurance
* safe feature boundaries

## Final Quality Bar

A locale strategy passes only if:

* the market angle is immediately understandable
* the hero visual shows an EPUB cover transformation
* before and after covers are clearly different
* workflow is simple and not technical-first
* privacy proof is visible and natural
* palette instructions are specific
* copy safe area is protected
* unsupported claims are avoided
* the app does not look like a generic image editor
* the app does not look like an ebook reader, downloader, cloud editor, or AI generator

## Global Recommended Sequence

Use this sequence unless a locale-specific reason requires adjustment:

1. Pain point / before-after transformation
2. Select, adjust, done
3. Preview / layout control
4. 100% on-device trust proof
5. Change any EPUB cover / before-after proof
6. Optional save-a-new-copy reassurance

## Current Global Direction

Current strongest global promise:

Change the cover of any EPUB file in seconds.

Current strongest global trust proof:

Everything is processed on your device. No accounts. No uploads.

Current strongest visual proof:

A weak, plain, or incorrect EPUB cover transformed into a polished, desirable cover.

Current strongest conversion logic:

Problem -> simple workflow -> control -> trust -> safe result.
