## Market Angle Decision

Market angle selected:
Change or fix an EPUB cover quickly with on-device processing and safe new-copy saving.

Why this angle should convert in this locale:
For hi-IN, simple Hinglish utility messaging is stronger than fully technical Hindi. The user should quickly understand the practical fix: EPUB cover अच्छा नहीं लग रहा, missing है, या wrong है, तो app से quickly replace किया जा सकता है.

Primary user desire:
खराब, missing, wrong या plain EPUB cover को जल्दी बदलना.

Secondary trust proof:
On-device processing, no account, no uploads, preview before saving, and saving a new copy instead of silently overwriting the original file.

What should NOT be the first screenshot:
Settings screen, empty library, technical file details, generic image editor UI, or privacy-only graphic without showing the cover transformation.

* optional support accent: #FFC48F

* subline color: #EED8C6

Reasoning:
For hi-IN, a dark utility base with warm orange accents can feel accessible, practical, and commercially clear. The warm accent should help guide attention, but the app should still feel trustworthy and focused.

The accent should be used only as controlled emphasis: before/after indicators, small trust markers, thin lines, subtle highlights near the final cover, or compact bullets.

General visual rules:
Keep the copy area dark, matte, and readable. The cover transformation must remain the main proof: weak/plain cover → better custom cover.

Avoid bright orange backgrounds, heavy glow behind text, too many decorative elements, AI sparkle effects, cloud/upload symbols, PDF visuals, Kindle-like device positioning, or anything that makes the app look like a generic photo editor.
