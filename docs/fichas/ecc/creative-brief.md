# EPUB Cover Changer — Creative Brief

## Purpose

Define the creative, commercial, visual, and localization strategy for Play Store listing assets of EPUB Cover Changer.

This brief exists to guide:

* Play Store copy generation
* feature graphic direction
* screenshot strategy
* regional localization
* golden listings
* bulk ficha generation
* strategy matrix generation
* conversion audits
* asset QA

This document is the strategic source of truth.

All generated fichas, golden listings, strategy matrices, conversion audits, and Play Store asset instructions must follow this brief.

## Product Identity

### Working product name

EPUB Cover Changer

### Category

EPUB utility / ebook library personalization.

### What the app really is

A focused utility that lets users replace the cover of an EPUB file with a new image, adjust it, preview the result, and save a new copy of the book.

### What the app is not

* Not a generic image editor
* Not a full EPUB editor
* Not an ebook reader
* Not an ebook store
* Not an official Amazon, Kindle, Kobo, Adobe, Apple Books, Google Play Books, or EPUB standards app
* Not a cover generator
* Not primarily an AI or template app
* Not a general book management platform
* Not a DRM tool
* Not an ebook downloader
* Not a format converter unless explicitly supported in the current build

## Core Positioning

### Main positioning statement

Change the cover of any EPUB file in seconds.

### Stronger practical interpretation

Fix bad, missing, ugly, plain, or incorrect EPUB covers quickly and privately.

### Functional truth

The app helps users:

* select an EPUB file
* choose a new cover image
* crop and align the image
* adjust the cover layout
* preview the result
* save a new copy of the EPUB with the updated cover

### Strategic balance

EPUB Cover Changer should be positioned as:

1. Practical problem-solving first
2. Simple cover customization second
3. Technical EPUB processing third

The user should understand the result before the technical details.

The app should not lead with file internals, EPUB structure, metadata language, or technical explanations unless the specific locale strategy requires more technical reassurance.

## Core Conversion Promise

### Primary conversion promise

Users can replace an EPUB cover quickly using their own image.

### Secondary conversion promise

The app gives enough control to crop, adjust, preview, and save the result without complicated desktop software.

### Trust proof

* 100% on-device processing
* no accounts
* no uploads
* preview before saving
* saves a new copy instead of silently overwriting the original
* focused workflow made specifically for EPUB covers

### Product Truth / Supported Feature Guardrails

The listing must stay aligned with what the app actually supports.

#### Safe promises

* Change an EPUB cover
* Replace the original EPUB cover
* Use a custom image
* Crop, align, and adjust the cover
* Preview the result
* Save a new copy of the EPUB
* Process files locally on the device
* Work with most EPUB readers

#### Risky promises

Do not over-emphasize these unless clearly verified in the current build:

* complete EPUB editing
* metadata editing
* batch processing
* AI cover generation
* template marketplace
* automatic cover search
* official compatibility with every EPUB reader
* support for DRM-protected books
* direct publishing features
* format conversion
* book repair beyond cover replacement

#### Important strategic truth

The app’s strongest promise is not:

Design a beautiful cover from scratch.

The strongest promise is:

I have an EPUB with a bad cover, and this app lets me fix it fast.

* organize digital libraries

* download public-domain or personal EPUB files

* want a simple mobile alternative to desktop ebook tools
* want to fix covers without using a full EPUB editor

### User mindset

The user is not searching for a complex EPUB editor.

They are searching for a simple way to change, fix, or improve an EPUB cover.

The user likely wants the result more than the feature list.

### Global visual baseline

The current ECC visual direction is dark, matte, practical, and direct.

Recommended global baseline:

* primary background: #101418
* secondary background: #1A2330
* accent: #4A78A8
* optional support accent: #6FA3D8
* optional warm accent: #D28A52
* headline color: #FFFFFF
* subline color: #C9D7E6
* bullet/support text color: #DCE6F2

### Why this baseline works

The charcoal + slate navy + steel blue direction feels more trustworthy, modern, and utility-tech than flat grayscale.

It preserves the strong dark look of the current assets while improving the feeling of privacy, control, and product quality.

The warm accent should mainly come from the replacement cover artwork, not from the full system background. This makes the “after” cover feel more desirable while keeping the app itself clean and reliable.

### Accent usage rule

The accent color must be restrained.

Use it for:

* subtle depth behind devices
* small trust cues
* controlled UI highlights
* low-intensity visual separation

Do not use it as:

* bright glow behind headlines
* large decorative blobs
* aggressive neon lighting
* dominant background color
* noisy gradient behind copy

## Image Use Rules

### Preferred cover content

Depending on region:

* polished illustration
* fantasy landscape
* cozy reading image
* personal photo
* pet photo
* travel memory
* minimal book-art style
* public-domain inspired cover
* aesthetic image that clearly improves the original cover

### Preferred “before” content

* plain white generic cover
* incorrect-looking minimal cover
* ugly placeholder cover
* missing-cover style
* rough sketch cover
* generic title-only layout
* visually weak cover with poor spacing

### Preferred “after” content

* polished, attractive, readable cover image
* warm or emotionally appealing image
* clearly better than the before cover
* strong enough to sell the value without needing explanation

### Avoid

* copyrighted recognizable book covers
* real publisher artwork
* fake Amazon/Kindle UI
* cluttered stacks of books
* tiny unreadable text
* AI generation claims
* covers that look too similar before and after
* visuals where the transformation is not obvious
* official brand logos
* images that imply the app provides books
* images that imply illegal downloading or DRM bypass
* PDF-like document visuals unless explicitly creating a separate PDF product asset

## Long Description Quality Standard

Long descriptions must be clear, practical, and conversion-focused.

### Target quality

A strong long description should include:

* opening hook
* immediate value proposition
* simple workflow
* feature benefits explained as outcomes
* privacy / local processing trust proof
* safe-copy reassurance
* natural ASO keywords
* closing motivation to install

### Recommended length

* Minimum acceptable: 700 characters
* Preferred range: 900–1600 characters
* Maximum: 4000 characters

If a generated long description is under 600 characters, it should usually be revised unless a specific store or locale constraint requires shorter copy.

### Tone rule

The long description should sound direct and useful, not poetic.

This app solves a concrete file problem.

Do not over-romanticize it.

### Description structure recommendation

Recommended long description structure:

1. Direct hook
2. Simple workflow
3. Feature bullets
4. Local/private trust proof
5. Use cases
6. Safe-copy reassurance
7. Focused closing line

## Short Description Quality Standard

Each locale should generate 3 candidates:

* SEO-first
* Conversion-first
* Balanced

The selected option must be chosen intentionally, not automatically.

### Rule

Do not default to “Balanced” every time.

Choose:

* SEO-first when discoverability is the main challenge
* Conversion-first when the problem/benefit is obvious
* Balanced only when it is clearly the strongest option

### Strong short description patterns

Safe structures:

* Change any EPUB cover in seconds
* Edit and replace EPUB covers easily
* Fix incorrect EPUB covers on your device
* Replace EPUB covers with your own image
* Change, crop, and save EPUB covers

Avoid vague structures:

* Make your library beautiful
* Create amazing covers
* Design better ebooks
* Manage your books easily
* Improve your reading experience

These are too broad and can misrepresent the app.
