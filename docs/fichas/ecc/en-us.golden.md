# EPUB Cover Changer - en-US Golden Play Store Listing

## Purpose

Position ECC for the US market as a fast, focused EPUB utility for readers who want to replace, fix, or improve the cover of an EPUB file without complicated editing software.

The listing should sell the outcome first:

Change the cover of any EPUB file in seconds.

The app should feel simple, useful, private, local, and trustworthy. The user should immediately understand that they can select an EPUB file, choose a new image, adjust the cover, preview the result, and save a new copy of the book.

The strongest conversion angle is not “ebook manager,” “reading app,” “AI cover generator,” “EPUB repair suite,” or “graphic design tool.” It is:

I have an EPUB with a bad, missing, incorrect, or plain cover, and I want to replace it quickly.

ECC should be presented as a direct EPUB cover replacement utility. It should not be positioned as a marketplace, reader, DRM tool, online converter, or general EPUB editor.

## Market Angle Decision

Market angle selected:
Fast EPUB cover replacement with precise local editing and safe save-as-copy behavior.

Why this angle should convert in this locale:
US users looking for this kind of app are likely trying to solve a very specific problem: an EPUB has the wrong cover, no cover, a plain cover, or a cover that makes their digital library look messy.

The listing should not over-explain EPUB internals. It should show a direct transformation: bad/plain cover → better cover. Then it should reinforce trust with on-device processing, no accounts, no uploads, and saving a new copy.

Primary user desire:
Replace a bad, missing, incorrect, or unattractive EPUB cover with an image they choose.

Secondary trust proof:
100% on-device processing, no accounts, no uploads, precise crop/layout control, preview before saving, and saving a new copy of the book.

What should NOT be the first screenshot:
A settings screen, technical file metadata, empty state, generic editor-only UI, privacy-only claim, or library view without showing the cover transformation.

The first screenshot must show the problem and the solution immediately.

## Regional Conversion Strategy

Primary user desire:
Fix or personalize EPUB covers quickly so the user’s ebook library feels cleaner, more organized, and more polished.

Best use cases to show:
Plain EPUB cover replaced with a polished cover, incorrect cover fixed, personal image adapted as an ebook cover, library view with the improved result, editor flow proving crop and layout control.

What to avoid:
Kindle/e-reader screensaver language, Kindle model compatibility, PDF cover language, AI cover generation claims, DRM implications, piracy-adjacent copy, ebook download concepts, marketplace visuals, or visuals that make the app look like a reading app.

Search/ASO terms to include naturally:
EPUB cover changer, change EPUB cover, edit EPUB cover, replace EPUB cover, ebook cover editor, EPUB editor, crop cover image, digital library, fix book cover, personalize ebooks.

## Visual System

Palette:

* primary background: #101418
* secondary background: #1A2330
* accent: #4A78A8
* optional support accent: #6FA3D8
* optional warm accent: #D28A52
* headline color: #FFFFFF
* subline color: #C9D7E6
* bullet/support text color: #DCE6F2

Reasoning:
For en-US, the palette should feel trustworthy, practical, modern, and premium. A charcoal-to-slate-navy base works better than flat grayscale because it keeps the current dark, high-converting look while adding a subtle tech/productivity feel.

The restrained steel-blue accent communicates reliability, control, privacy, and utility. The warm accent should mainly come from the replacement cover artwork itself, not from the full UI system. This lets the “after” cover feel more emotional and desirable while the overall listing remains clean and trustworthy.

General visual rules:
Use a premium dark utility-tech visual system. The tone should feel clean, capable, dependable, and direct.

The visual language should prioritize transformation:
bad/plain cover → polished cover.

Keep typography bold, readable, and high contrast. Keep the copy area dark and stable. Blue accent light may be used for controlled depth behind devices or cover mockups, but never as a bright glow behind text.

Avoid:

* flat mid-gray backgrounds
* overly warm brown/orange backgrounds
* heavy glow behind headlines
* bright neon blue
* decorative clutter
* playful gradients that reduce trust
* Kindle/e-reader screensaver visuals
* PDF visuals
* book marketplace visuals
* reading app concepts
* cloud/upload imagery
* AI generation cues

## Notes / Assumptions

* This golden ficha uses the current high-converting ECC visual direction: dark background, bold white headline, clean gray-blue subline, and clear before/after cover transformation.
* The en-US palette has been adjusted from neutral black/gray toward charcoal + slate navy + steel blue to improve trust, clarity, and utility-tech perception.
* ECC changes EPUB covers. It should not be described as a Kindle screensaver app, PDF cover maker, ebook reader, marketplace, DRM tool, or ebook downloader.
* The app should not imply that it provides books, removes DRM, edits EPUB text content, converts formats, or generates AI artwork.
* “Save a new copy” is an important trust phrase because it reduces perceived risk.
* “100% on your device,” “No accounts,” and “No uploads” should remain visible trust claims because they match the app’s utility positioning.
* Warm colors should come primarily from the replacement cover image, not from the UI/background system.
* The current en-US assets are strong because they are direct, readable, and instantly explain the product. Future variants should preserve that clarity instead of becoming more decorative.
* This golden ficha is a quality benchmark for Codex generation, not a template to copy literally across locales.
